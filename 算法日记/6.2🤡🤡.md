## 374.前k个元素
#栈 #队列 
![[Pasted image 20260602230636.png]]


### 核心思想
==利用小顶堆储存从小到大存储元素，然后存储大于k之后把队列顶的踢出队列==
要用哈希表存元素和出现次数
### 代码部分
```cpp
#include <vector>
#include <unordered_map>
#include <queue>
using namespace std;

class Solution {
public:
    vector<int> topKFrequent(vector<int>& nums, int k) {
        // 1.统计每个数字出现频次
        unordered_map<int, int> freq;
        for (int num : nums) {
            freq[num]++;
        }

        // 2.小根堆：pair<数值,频次>，频次小的在堆顶
        auto cmp = [](pair<int, int>& a, pair<int, int>& b) {
            return a.second > b.second;
        };
        // 优先队列：底层容器+自定义比较函数实现小顶堆
        priority_queue<pair<int, int>, vector<pair<int, int>>, decltype(cmp)> min_heap(cmp);

        // 3.遍历哈希表，堆中只保留前k个高频元素
        for (auto& entry : freq) {
            min_heap.push(entry);
            if (min_heap.size() > k) {
                min_heap.pop(); // 删掉当前频次最小的
            }
        }

        // 4.取出堆内所有答案
        vector<int> result;
        while (!min_heap.empty()) {
            result.push_back(min_heap.top().first);
            min_heap.pop();
        }
        return result;
    }
};
```
### 复杂度分析
O(nlogk)

#栈 
### 1.区别cmp
**stack中**
```cpp
auto cmp = [](pair a,pair b){return a.second > b.second;}
// 从小到大
// 比较规则：a.second > b.second 成立，a去下面
// 堆顶永远存当前频率最小的元素
```
**常见sort中**
```cpp
bool cmp(int a,int b)
{
return a>b
}
//从大到小
//成立，a去上面
```
### 2.stack与queue的区别
1. **操作原则**：Stack 是**后进先出**（LIFO），Queue 是**先进先出**（FIFO）。
    
2. **插入删除位置**：Stack 只在**同一端**（栈顶）插入和删除；Queue 在**两端**操作（队尾插入，队首删除）。
    
3. **典型应用**：Stack 用于函数调用、括号匹配；Queue 用于任务调度、广度优先搜索。
### for循环遍历容器
-  for(int x : arr) ：拷贝元素，修改x不影响原数组

-  for(int &x : arr) ：引用，可修改原数组元素

-  for(const int &x : arr) ：常引用，只读、效率最高
